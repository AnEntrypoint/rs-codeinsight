## Digest-churn root-cause pattern: check the actual caller before assuming the crate under audit is the producer

When tasked to root-cause a shared-format churn bug (digest/hash/manifest non-determinism) inside crate A because A 'likely produces' the format: verify by grep for the suspected function's callers across every consuming repo BEFORE editing. A can export a plausibly-matching function that is dead code / superseded by an independent reimplementation in the actual consumer (e.g. a host/orchestrator crate holding its own git-HEAD+diff-based digest). Fix the real bug found (mtime/wall-clock/working-tree-state hashed into a digest breaks f-compose-f-equals-f) regardless, since a public API with a live non-determinism bug is worth fixing even if unused today -- but report the causal chain accurately: 'this crate had a latent bug, but was not the producer of the observed churn' is a different finding than 'root cause confirmed and fixed', and conflating them misdirects the next investigator toward the wrong repo.
